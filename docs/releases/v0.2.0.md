# SnipDock v0.2.0

## 版本定位

`v0.2.0` 聚焦安装与发布体验，让普通用户更容易安装、启动和卸载 SnipDock。

SnipDock 仍然是 Windows 本地轻量片段管理器：数据保存在本地，不上传云端，不自动执行命令。

## 主要变化

- 新增 Inno Setup 安装包脚本：`installer/SnipDock.iss`。
- 安装包按当前用户安装，不需要写入全局 Program Files。
- 安装包支持开始菜单快捷方式。
- 安装包可选创建桌面快捷方式。
- 卸载程序只移除安装目录中的程序文件，不删除用户选择的数据目录。
- 关于页面显示当前版本号。
- 关于页面新增“检查更新”入口，打开 GitHub Releases 页面。
- README 增加安装包构建和安装说明。

## 运行要求

- Windows
- .NET 9 Desktop Runtime

当前安装包和 zip / 便携包均为 framework-dependent 形式，不内置 .NET Desktop Runtime。

## 下载和运行方式

### 安装包

下载并运行：

```text
SnipDock-Setup-v0.2.0.exe
```

安装完成后，可以从开始菜单启动 SnipDock。如果安装时勾选桌面快捷方式，也可以从桌面启动。

### Zip / 便携包

解压后运行：

```text
SnipDock.App.exe
```

## 构建发布产物

```powershell
dotnet publish src/SnipDock.App/SnipDock.App.csproj -c Release -r win-x64 --self-contained false -o .\publish\SnipDock
```

## 构建安装包

安装 Inno Setup 后执行：

```powershell
ISCC.exe .\installer\SnipDock.iss
```

安装包输出目录：

```text
dist\installer\
```

## 数据与隐私

- 用户数据保存在首次启动时选择的本地目录。
- 主数据文件仍为 `prompts.json`，暂不强制改名。
- 自动备份位于用户数据目录下的 `backups\`。
- 日志不应记录条目正文和剪贴板内容。
- 命令类条目只复制，不执行。
- 卸载 SnipDock 不会删除用户选择的数据目录。

## 从 PromptShelf 迁移

- 继续兼容旧 `%APPDATA%\PromptShelf\bootstrap.json`。
- 迁移后会写入 `%APPDATA%\SnipDock\bootstrap.json`。
- 不删除旧 PromptShelf 配置。
- 不移动用户自定义数据目录。

## 已知限制

- 安装包暂不内置 .NET Desktop Runtime。
- 暂不提供自动更新器。
- “检查更新”只打开 GitHub Releases 页面，不自动下载或覆盖安装。
- 搜索范围仍为标题和标签，不搜索正文内容。
- 命令条目只复制，不执行。
