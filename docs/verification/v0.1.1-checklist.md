# SnipDock v0.1.1 Manual Verification Checklist

本清单用于 v0.1.1 beta 稳定性修复阶段的手动验收。开发期只同步 Debug exe；Release exe 或发布包只在本清单通过后再生成和分发。

## 1. 构建与测试

- [ ] `dotnet build SnipDock.sln` 通过。
- [ ] `dotnet test SnipDock.sln` 通过。
- [ ] Debug exe 已同步到当前代码。
- [ ] 未更新 Release 产物，除非准备正式发布。

## 2. 仓库与数据安全

- [ ] `bin/`、`obj/`、`publish/`、`logs/`、`backups/` 未提交。
- [ ] `prompts.json`、`prompts.json.bak`、`settings.json`、`bootstrap.json` 未提交。
- [ ] 未提交真实用户数据、真实日志、真实剪贴板内容或本机私有路径。
- [ ] `GOAL.md` 保持本地忽略，不作为公开仓库文档提交。

## 3. 启动与常驻

- [ ] 首次启动缺少数据目录时能打开配置窗口。
- [ ] 选择数据目录后可以进入主界面。
- [ ] 重复启动时只保留单实例，并提示已有 SnipDock 在运行。
- [ ] 系统托盘图标显示为 SnipDock。
- [ ] 托盘左键可以显示 / 隐藏管理面板。
- [ ] 托盘右键菜单可以打开数据目录、日志目录、备份目录和设置。
- [ ] 退出后托盘图标消失，无残留。

## 4. 开机自启

- [ ] Debug 开发目录下启用开机自启时会提示不建议启用。
- [ ] Release publish 或正式发布目录下不会被误判为开发调试目录。
- [ ] 启用自启后 HKCU Run 项为 SnipDock。
- [ ] 自启路径包含双引号和 `--startup` 参数。
- [ ] 关闭自启后 SnipDock 自启项被删除。
- [ ] 不生成新的旧 旧版应用 自启项。

## 5. 核心工作流

- [ ] 新建 Prompt、命令、代码片段、笔记条目可保存。
- [ ] 编辑条目后内容、类型和标签保存正确。
- [ ] 删除条目前有确认提示。
- [ ] 复制条目只写入剪贴板，不执行命令。
- [ ] 复制后 Toast 可读且会自动消失。
- [ ] 可选“复制后自动隐藏面板”行为正常。
- [ ] 标题搜索正常。
- [ ] 标签搜索正常。
- [ ] 搜索不匹配正文内容。
- [ ] 类型筛选和标签筛选正常。
- [ ] 收藏、置顶、最近使用和使用次数正常更新。

## 6. 语言与文案

- [ ] 首次默认语言根据系统语言选择。
- [ ] 设置中可以切换中文 / English。
- [ ] 管理面板主文案跟随语言切换。
- [ ] 类型筛选的 All / 全部显示正确。
- [ ] 标签筛选的 All tags / 全部标签显示正确。
- [ ] 托盘菜单跟随语言切换。
- [ ] 悬浮球右键菜单跟随语言切换。
- [ ] 配置数据目录窗口跟随系统语言或已保存语言显示。
- [ ] 不把产品主品牌显示为 旧版应用。

## 7. 数据、备份、导入导出

- [ ] 主数据文件仍为 `prompts.json`。
- [ ] 安全备份 `prompts.json.bak` 正常生成。
- [ ] 自动备份目录 `backups\` 正常生成。
- [ ] 手动备份可以创建。
- [ ] 从备份恢复前会保护当前数据。
- [ ] JSON 导出可生成文件。
- [ ] JSON 导入不会破坏现有数据。
- [ ] 切换 StoragePath 不会自动移动或改名用户数据目录。
- [ ] 旧 旧版应用 bootstrap 兼容迁移仍可用，且不会删除旧配置。

## 8. 视觉与窗口

- [ ] Light / Dark 主题正常。
- [ ] Accent 配色切换正常。
- [ ] 悬浮球可拖动，位置能保存。
- [ ] 重置悬浮球位置正常。
- [ ] 管理面板右侧详情、编辑状态和空状态显示正常。
- [ ] Toast 不出现空白矩形。

## 9. 发布前结论

- [ ] 以上项目通过后，可以考虑生成 Release publish。
- [ ] Release notes、README、版本号和发布包内容一致。
- [ ] 如果发布 beta，需要在 GitHub Release 标记为 pre-release。
